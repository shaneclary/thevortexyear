Specifically: n must be a positive integer (closure condition); n must be coprime with 1 (always true; gives knot); n value beyond this is partly a foundation specification choice. The canonical specification fixes n; the underlying mathematical structure rigidly constrains which n values are valid; the foundation chooses among valid options. Rigidity is structural; specification is canonical-within-constraints.*

> *Three layers in the defence.*
>
> *Layer one — closure condition forces integer n.* *(p, q) torus knot closes only when p, q are integers*; *n must be integer; not arbitrary real number*. *Mainstream topology forces this.*
>
> *Layer two — coprime condition.* *gcd(p, q) = 1 for knot (rather than link)*; *with p = 1, gcd(1, n) = 1 for any integer n*; *condition automatically satisfied*. *Layer-1 mathematics.*
>
> *Layer three — foundation chooses among valid options.* *n ∈ {1, 2, 3, ...} all give valid (1, n) torus knots; foundation specifies one as canonical*; *Rodin Coil = (1, n_canonical)*. *Foundation specification within mathematical constraints.*
